# Parquet Reader 사용 가이드

본 프로그램은 대용량 Parquet 데이터 파일(`.parquet`, `.pq`)의 스키마와 레코드를 신속하게 확인하고 조회할 수 있는 가벼운 데스크톱 GUI 뷰어입니다. 데이터 분석 패키지(Pandas, NumPy 등)의 임포트 오버헤드를 배제하고, PyArrow 엔진만을 직접 호출하여 오프셋 슬라이싱 방식으로 고속 렌더링하도록 구현되었습니다.

---

## 🌟 주요 기능

*   **메모리 최적화 렌더링**: 대량의 데이터셋 로드 시, 메모리 복사 없이 화면에 보이는 레코드 단위(`PyArrow.Table.slice`)로 가상 페이징 처리하여 즉각 구동됩니다.
*   **파일 드래그 앤 드롭 지원**: 실행된 메인 윈도우 인터페이스 위로 로컬 Parquet 파일을 마우스 드래그하여 드롭하는 것만으로 데이터를 로드할 수 있습니다.
*   **컴퓨팅 기반 다기능 필터**: `pyarrow.compute` 모듈을 연동하여 지정 컬럼에 대한 비교 연산(`==`, `!=`, `>`, `<`, `contains`, `starts with`, `ends with`) 및 Null 예외 처리를 지원합니다.
*   **연결 프로그램 및 아이콘 연동**: 사용자의 윈도우 환경에 부합하도록 전용 데스크톱 실행 파일 등록 및 파일 탐색기 확장자 아이콘 뷰 매핑을 지원합니다.
*   **최근 파일 히스토리 캐싱**: 로컬 파일 접근 이력을 8개까지 저장 및 캐싱하여, 더블클릭을 통해 신속하게 이전 데이터셋에 재접근할 수 있습니다.

---

## 🚀 환경 설정 및 윈도우 파일 연결 방법 (최초 1회)

배치 실행 시 발생하는 도스 콘솔 창(`cmd.exe`)의 노출 없이, 윈도우 기본 셸 단에서 무음으로 연동되도록 등록하는 절차입니다.

1.  프로젝트 디렉토리 내에 위치한 **`register_association.bat`** 파일을 실행합니다.
2.  사용자 단독 레지스트리 영역(`HKCU\Software\Classes`)에 확장자(`.parquet`, `.pq`) 정의 및 전용 실행 파일(`ParquetReader.exe`) 등록이 자동으로 완료됩니다.
3.  성공 메시지 확인 후 창을 닫습니다.
4.  로컬 Parquet 파일을 마우스 더블클릭한 후 **"이 파일을 어떻게 여시겠습니까?"** 대화상자가 표시되면:
    *   앱 목록에서 **`Parquet Reader`**를 지정합니다.
    *   **"항상 이 앱을 사용하여 .parquet 파일 열기"** 항목을 선택하고 **[확인]**을 누릅니다.
5.  이후 확장자 연결이 상시 고정되며, 셸 기동 번쩍임 없이 어플리케이션이 실행됩니다.

---

## 💡 기본 사용법

### 1. 셸 연결 열기
연결 프로그램 설정이 완료된 뒤, 탐색기 상의 Parquet 파일을 마우스 더블클릭하면 윈도우 백그라운드로 즉각 구동 및 로드됩니다.

### 2. 수동 기동 및 드롭 로딩
1.  **`dist\ParquetReader\ParquetReader.exe`** 실행 파일을 직접 더블클릭하여 빈 화면의 뷰어를 단독 구동합니다.
2.  탐색기 내의 타겟 Parquet 파일을 마우스 드래그하여 실행된 테이블 뷰 보디 위로 드롭하면 데이터 렌더링이 수행됩니다.

### 3. 필터 쿼리 연산 및 데이터셋 익스포트
*   **데이터 필터링**: 상단 필터 바 영역에서 컬럼을 특정하고, 조건 연산자를 지정한 뒤 기준 값을 입력하여 `Apply Filter`를 클릭하거나 엔터키를 입력합니다.
*   **포맷 변환 내보내기**: 현재 필터링되어 화면에 표시되고 있는 데이터 프레임을 다른 포맷으로 영구 보관하려면 하단 좌측의 `Export Dataset...` 버튼을 눌러 **CSV, JSON, 혹은 신규 Parquet 포맷**으로 출력할 수 있습니다.

## ⚙️ 문제 해결 및 자주 묻는 질문 (FAQ)

**Q. 대용량 파일의 레코드가 화면 내에 전부 표시되지 않습니다.**
*   A. 인터페이스의 과부하 방지 및 탐색 스크롤 최적화를 위하여 데이터가 페이지 단위로 분할 로드됩니다. 하단 중앙의 화살표 패널(`|<`, `<`, `>`, `>|`)로 페이지 이동이 가능하며, `Page Size` 옵션 조절을 통해 로드 행 개수 한계(최대 2,000줄)를 동적으로 확장할 수 있습니다.

**Q. 기존 수동 설정된 셸 환경 또는 이전 배치 프로그램으로 연결 우선순위가 오작동합니다.**
*   A. 윈도우 OS의 기본 앱 고정 메커니즘 충돌 상태입니다. 대상 파켓 파일 우클릭 후 **`속성(Properties)`** -> 일반 탭 내 연결 프로그램의 **`변경(Change...)`** 버튼을 눌러, 어플리케이션 풀에서 **`Parquet Reader`**를 우선 순위로 다시 확정해 주시면 즉각 복원됩니다.

**Q. 로컬 테스트 및 동작 검증을 위한 예비 샘플이 존재하지 않습니다.**
*   A. 메인 레이아웃 우측 상단의 **`Generate Sample Data`** 기능을 트리거하면, 1만 행 규모의 유의미한 가상 매출 로그 원시 Parquet 파일이 1초 내로 프로젝트 내 로컬 데이터 폴더에 쓰여지며 자동 파싱 테스트 모드로 진입합니다.
